# DataTester 子方案：DataTester-Game

## 业务背景

将游戏相关数据传输给专门分析游戏的第三方公司。核心传输原则：
- **明文传输**：字段类型为"游戏行为日志"的数据
- **匿名化传输**：字段类型为"账号ID"或"设备ID"的数据
- **不传输**：字段类型为"其他"的数据

## 输入字段说明

| 列 | 列名 | 含义 | 操作 |
|----|------|------|------|
| A | 字段名* | 字段英文标识符 | 原样保留 |
| B | 字段类型* | 数据类型（bigint/string/int等） | 原样保留 |
| C | 字段描述 | 字段中文说明 | 原样保留 |
| D | 字段示例值 | 样本值 | 原样保留 |
| E | 传输形式（法务） | 法务标注的传输形式 | 原样保留，用于 K 列对比 |
| F | 事件描述（合并） | 该字段所属的埋点事件（可多个，逗号分隔） | 原样保留，作为判断主依据 |
| G | 数据定义 | **待填写**：数据分类（如关卡、副本日志） | 填写 |
| H | 字段类型 | **待填写**：字段归类（账号ID / 设备ID / 游戏行为日志 / 其他） | 填写 |
| I | 传输方式 | **待填写**：按业务逻辑确定的传输方式 | 填写 |
| J | 审阅人 | 审阅人姓名 | 原样保留 |
| K | 备注 | **待填写**：若 I ≠ E，按判断依据写备注（详见判断标准第4步） | 填写 |

## 判断标准

### 主流程
1. 根据 F 列"事件描述"，对照下方完整映射表，找到匹配的"事件描述类型"所在行
2. 将该行的规则文件 B 列（数据定义）→ 填入 G 列，A 列（字段类型）→ 填入 H 列
3. 根据 H 列值，按下方"传输方式推导规则"确定 I 列
4. 对比 I 列与 E 列：一致则 K 列留空；不一致则按以下优先级写备注：
   - F 列有效 + A 列字段名也参与了判断：`不一致：由事件描述「{关键事件}」和字段名「{A列字段名}」判断为{H列内容}，传输方式{I列内容}，法务标注为{E列内容}`
   - 仅 F 列事件描述即可判断：`不一致：由事件描述「{关键事件}」判断为{H列内容}，传输方式{I列内容}，法务标注为{E列内容}`
   - F 列为空或完全无助于判断（仅凭字段名判断）：`不一致：由字段名「{A列字段名}」判断为{H列内容}，传输方式{I列内容}，法务标注为{E列内容}`

### 传输方式推导规则（由 H 列决定 I 列）

| H 列（字段类型） | I 列（传输方式） |
|----------------|----------------|
| 游戏行为日志 | 明文传输 |
| 账号ID | 匿名化传输 |
| 设备ID | 匿名化传输 |
| 其他 | 不传输 |

### 业务覆盖原则（优先于字面匹配）
- **账号ID**：标识特定用户/玩家/角色的唯一标识符（user_id、account_id、role_id、UID、OpenID 等），以及用户登录 IP → 匿名化传输。字段示例值若为十位数数值，可初步判断为游戏账号ID
- **设备ID**：识别用户设备的唯一标识（IDFA、IDFV、Android ID、OAID、IMEI、GUID、UUID 等）→ 匿名化传输
- **设备基础信息**（设备型号、系统版本、CPU、内存、分辨率、运营商、网络类型、WiFi 等）→ 不传输，归属"其他"
- **纯个人身份信息**（手机号、Email、位置信息等）→ 不传输，归属"其他"

## 字段名辅助判断规则

当 F 列事件描述不足以判断时，可参考 A 列（字段名*）的英文命名模式辅助判断。

### 游戏对战性能指标 → 游戏行为日志 / 关卡、副本日志 / 明文传输

以下字段名模式在游戏对局中采集，反映玩家操作体验，属于游戏行为数据：

**核心操作卡顿指标**
- `movelag`：移动卡顿次数（指令发出后服务器未在阈值内确认的次数）
- `skilllag`：技能卡顿次数（按键到技能生效之间发生明显延迟的次数）
- `opertimes`：操作总次数，可用于计算 APM
- `operate350`：延迟 ≥ 350ms 极端网络环境下的操作次数

**网络延迟与丢包统计**
- `incominglagnum`：下行网络卡顿总数
- `outgoinglagnum`：上行网络卡顿总数
- `outgoingping50ms` / `outgoingping100ms` / `outgoingpingover1...`：上行延迟区间采样次数
- `incomingping50ms` / `incomingping100ms` / `incomingpingover1...`：下行延迟区间采样次数
- `incoming265` / `incoming530` / `incoming1000` / `incoming3000`：下行异常延迟阈值阶梯计数（单位 ms）

**网络协议传输指标**
- `switchudpnum` / `switchtcpnum`：切换 UDP / TCP 协议次数
- `udpframenum` / `tcpframenum`：各协议传输数据帧总数
- `totalclientsendpack`（及同前缀字段）：客户端发送总数据包数

**对局状态标记**
- `isinvalidbattle`：无效对局标记（0/1），用于标识作废对局

**角色与分区标识**
- `zoneid` / `zone_id`：游戏分区ID → 游戏行为日志 / 游戏角色创建日志 / 明文传输
- 字段名或字段描述中含"分区ID"、"分区"、"区服"等词时，同上判断

### 不确定字段的处理

- `clientparam6` / `clientparam7` / `clientparam8`：含义不固定的预留参数，需结合 C 列字段描述和 D 列示例值综合判断；如无明确指向，默认归为 其他 / 不传输
