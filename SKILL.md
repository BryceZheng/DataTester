# DataTester — 通用 Excel 数据校验工具

## 脚本路径

| 脚本 | 位置 | 用途 |
|------|------|------|
| `read_excel.py` | `~/.claude/skills/DataTester/read_excel.py` | 读取 Excel 指定行范围，输出 JSON |
| `write_excel.py` | `~/.claude/skills/DataTester/write_excel.py` | 将 JSON 数据按模板写入 Excel |

---

## 阶段 1：读取并分析输入文件

### 目标
理解输入 Excel 的字段结构，与用户对齐每列的含义。

### 执行步骤

**步骤 0：检测 Python 命令（跨平台）**

在执行任何 Python 脚本之前，先逐个尝试以下命令，找到第一个成功返回版本号的，之后整个会话所有 `python3` 均替换为该命令：

1. 运行 `python3 --version`
2. 若失败，运行 `python --version`
3. 若仍失败，运行 `py --version`

记录可用命令（下文统称 `<py>`），macOS/Linux 通常为 `python3`，Windows 通常为 `python` 或 `py`。

---

**步骤 1：询问用户选择模式**

完成 Python 检测后立即询问：

> "请选择本次运行模式：
> 1. 创建新方案（逐步配置字段说明、判断标准和输出模板）
> 2. 使用已有子方案（加载上次保存的配置，跳过配置环节）"

**若用户选择「1. 创建新方案」**，直接进入步骤 1。

**若用户选择「2. 使用已有子方案」**，则：

```bash
<py> -c "import os,glob; d=os.path.join(os.path.expanduser('~'),'.claude','skills','DataTester','presets'); ps=[os.path.splitext(os.path.basename(p))[0] for p in glob.glob(os.path.join(d,'*.md'))]; print('\n'.join(ps))"
```

- 若目录不存在或为空，告知用户"暂无已保存的子方案"，自动切换为创建新方案流程。
- 若有子方案，列出可用方案（去掉 `.md` 后缀），询问：

> "可用子方案如下：
> 1. <方案名1>
> 2. <方案名2>
> ...
>
> 请输入要加载的方案名称或序号："

收到用户选择后：
1. 读取 `~/.claude/skills/DataTester/presets/<方案名>.md`
2. 向用户展示该方案的字段说明、判断标准、输出模板说明
3. 询问：

> "以上是「<方案名>」的配置，请提供本次要处理的文件路径：
> 1. 输入 Excel 文件路径：
> 2. 输出模板 Excel 路径（方案默认：`<template_path>`，如沿用请回复「是」，否则请输入新路径）："

4. 获取路径后，**跳过阶段 1-3，直接进入阶段 4（Demo 试跑）**。

## 阶段 2：收集判断标准

### 目标
获取本次任务的具体业务规则，确保理解无误。

### 执行步骤

**步骤 1：询问提供方式**

> "请选择判断标准的提供方式：
> 1. 直接在对话中描述（文字说明、规则列表等均可）
> 2. 使用判断规则模板文件（DataTester判断规则模板.xlsx，结构化填写，Claude 精准读取）
>    - 已有模板文件 → 直接提供文件路径
>    - 尚无模板文件 → 可为您在桌面生成空白模板，填写后再提供路径"

---

**若用户选择「1. 直接描述」**：

> "请描述本次数据校验的判断标准。可以是文字说明、规则列表、条件映射表，或任何你觉得方便的形式。"

收到后以结构化方式复述：

```
我理解本次的判断标准如下：
1. [规则1]：[具体说明]
2. [规则2]：[具体说明]
...
特殊情况处理：[如有]
```

> "以上是我对判断标准的理解，请确认是否正确？"

⚠️ 用户确认后进入阶段 3。

---

**若用户选择「2. 使用判断规则模板文件」**：

**2a. 已有模板文件**：用户提供路径后，读取（`--header-row 4` 跳过前三行标题/说明，直接用第 4 行作为列名）：

```bash
<py> ~/.claude/skills/DataTester/read_excel.py \
  "<模板文件路径>" 5 999 --header-row 4
```

**2b. 尚无模板文件**：询问：

> "桌面尚无判断规则模板，是否现在为您生成？（将生成至 `~/Desktop/DataTester判断规则模板.xlsx`）"

若用户同意，执行：

```bash

```

> "模板已生成，请打开填写后告知我文件路径，我再继续读取。"

⚠️ 等待用户填写完毕并提供路径后，再执行 2a 的读取步骤。

**2c. 读取并解析模板**：从返回 JSON 中提取数据行（跳过标题行、说明行和表头行，即从第 5 行起的非空行），解析各列：

| 列 | 含义 |
|----|------|
| 适用字段 | 输入 Excel 中对应的列（如 A列、C列） |
| 判断条件 | 触发该规则的条件描述 |
| 输出字段 | 满足条件后写入的输出列名 |
| 输出值 | 写入的具体内容 |
| 优先级 | 数字越小越优先，冲突时依此裁定 |
| 备注 | 补充说明 |

向用户展示摘要确认：

> "已从模板读取到 X 条判断规则：
> - 规则1：[适用字段] [条件] → 输出 [字段]=[值]
> - 规则2：…
>
> 请问还有需要补充或修改的地方吗？确认后我们继续阶段 3。"

⚠️ 用户确认后正常进入阶段 3（输出模板分析仍由 Claude 读文件自动完成）。

## 阶段 5：全量处理（分批 + 双重核验）

### 目标
对全量数据进行高质量的分批处理，每批三遍核验，最终合并输出。

### 执行步骤

**步骤 1：获取总行数**

```bash
<py> ~/.claude/skills/DataTester/read_excel.py \
  "<输入文件路径>" count
```

计算需要处理的批次数（每批 20 行数据，即行 2-21 为第 1 批，22-41 为第 2 批，以此类推）。

**步骤 2：逐批处理**

对每一批（设为第 N 批，起始数据行为 `start = 2 + (N-1)*20`，结束行为 `start + 19`）：

**2a. 读取本批数据**
```bash
<py> ~/.claude/skills/DataTester/read_excel.py \
  "<输入文件路径>" <start> <end>
```

**2b. 第 1 遍判断**：按判断标准逐行处理，生成初始结果 JSON。

**2c. 第 2 遍核验（快速）**：重新阅读本批数据和初始结果，检查：
- 是否有遗漏的判断条件？
- 是否有与标准不符的结果？
- 边界情况是否处理正确？
  
标记所有发现的问题并修正。

**2d. 第 3 遍核验（快速）**：再过一遍，确认修正无误，以最终结论为准。

**2e. 写入批次 JSON 到 scratchpad**：

用 Write 工具将本批结果写入 `<scratchpad>/batch_<NNN>.json`（NNN 为三位数批次号，如 001、002），格式与 demo_output.json 完全一致：

```json
{
  "headers": ["输出列1", "输出列2", ...],
  "rows": [
    {"输出列1": "值", "输出列2": "值", ...},
    ...
  ]
}
```

**2f. 进度报告**：

从 read_excel.py 返回的 JSON 中取实际的 `start_row` 和 `end_row` 字段（已由脚本 clamp 到文件真实范围），用这两个值报告：

> "第 N 批（第 X-Y 行）处理完成，共 M 批。"

**步骤 3：合并所有批次**

所有批次处理完成后，用 Read 工具依次读取每个 `batch_001.json`、`batch_002.json`……，将各文件的 `rows` 数组拼接为一个列表，保留第一个文件的 `headers`，组成如下结构后用 Write 工具写入 `<scratchpad>/final_output.json`：

```json
{
  "headers": ["输出列1", "输出列2", ...],
  "rows": [ ...所有批次的rows合并... ]
}
```

**步骤 4：写入最终 Excel**

```bash
<py> ~/.claude/skills/DataTester/write_excel.py \
  "<模板文件路径>" \
  "<scratchpad>/final_output.json" \
  "<输入文件所在目录>/final_output_<YYYYMMDD>.xlsx"
```

**步骤 5：完成报告**

> "✅ 全量处理完成！
> 
> - 处理行数：X 行
> - 批次数：Y 批
> - 输出文件：`<路径>`
> 
> 请打开文件核查。如发现任何问题请告知，可以针对具体行重新处理。"

## 注意事项

- **JSON 写入**：始终先将 JSON 用 Write 工具写入 scratchpad 文件，再将该**文件路径**作为参数传给 Python 脚本（Python 脚本内部自行 `open()` 读取）；不要在命令行内嵌 JSON 字符串，避免中文引号或特殊字符破坏解析
- **scratchpad 路径**：取系统 prompt 中 `IMPORTANT: Always use this scratchpad directory` 后面给出的完整路径，不要自行推断或拼接
- **输出文件命名**：使用 `final_output_YYYYMMDD.xlsx` 格式，避免覆盖之前的结果
- **核验重点**：每批第 2、3 遍核验时，特别关注首行、末行、含空值的行、边界值行
- **Python 命令**：所有脚本调用均使用步骤 0 检测到的 `<py>` 命令，而非硬编码 `python3`
